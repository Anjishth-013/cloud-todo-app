Just built my first serverless cloud application from scratch! 

𝗪𝗵𝗮𝘁 𝗜 𝗯𝘂𝗶𝗹𝘁:
A full-stack Cloud To-Do app with zero server management — add, complete, and delete tasks, all stored permanently in the cloud.

𝗧𝗲𝗰𝗵 𝘀𝘁𝗮𝗰𝗸:
 React.js — Frontend UI with hooks, state management, and fetch API
 AWS Lambda (Python 3.11) — Serverless backend handling all CRUD/GPPD logic
 AWS API Gateway — REST API exposing Lambda via public HTTP endpoints
 AWS DynamoDB — NoSQL cloud database storing todos as JSON documents
 AWS IAM — Roles & policies controlling Lambda's access to DynamoDB
 AWS CloudWatch — Logging Lambda executions and monitoring errors
 AWS CLI — Deployed and configured everything from the terminal

𝗛𝗼𝘄 𝗶𝘁 𝘄𝗼𝗿𝗸𝘀:
React UI → API Gateway → Lambda (Python) → DynamoDB
One Lambda function handles GET, POST, PUT, and DELETE — no traditional server needed.

𝗪𝗵𝗮𝘁 𝗜 𝗹𝗲𝗮𝗿𝗻𝗲𝗱:
→ Setting up AWS from scratch — IAM users, access keys, CLI configuration
→ Writing Python Lambda functions for full CRUD operations using boto3
→ Configuring API Gateway with AWS_PROXY integration to trigger Lambda
→ Creating DynamoDB tables and querying them with scan() and put_item()
→ Managing IAM roles and attaching policies for secure service communication
→ Deploying everything via AWS CLI — no console clicking, pure commands
→ Enabling CORS so the React frontend can talk to the API without errors
